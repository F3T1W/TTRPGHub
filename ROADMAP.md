## Северная звезда

Сделать так, чтобы русскоязычному игроку/мастеру НРИ не нужно было уходить на сторонние
сайты ни для чего: вкат в хобби, изучение правил, билдостроение, поиск пати, ведение игр.
Один сайт закрывает весь цикл от «первый раз слышу про D&D» до «веду кампанию третий год».

Основная система — **Pathfinder 2e**. D&D 5e — второстепенная, черновая (см. фазу S).

## Актуальное состояние (ревизия 2026-07-15)

**Замена Foundry VTT для PF2e — закрыта.** Можно проводить живую сессию без VTT на удалённом
рабочем столе: карта (сетка/туман/стены/свет/сцены), бой (MAP, степени успеха, инициатива,
Dying/Wounded/Unconscious, ауры со спасброском), PF2e-лист персонажа за столом (спеллкастинг,
атаки, ресурсы класса, Hero Points), макросы в песочнице (личные + расшаренные, несколько
страниц хотбара), rule elements engine, компаньоны/транспорт, крафтинг, PFS chronicle sheets,
NPC-генератор, импорт купленных PDF-приключений в приватный контент сессии.

Полная история закрытых фаз A–Q (справочник правил, билдер, онбординг, тикеты, оффлайн-игра,
модерация, вся замена Foundry по шагам, импорт приключений, полный паритет фичи-в-фичи, объём
контента, полировка) — см. `git log`/предыдущие версии файла. Ключевое архитектурное решение,
на котором всё это построено — **системная независимость справочника**:

```
GameSystem (справочник систем: "dnd5e", "pf2e", "custom:<slug>")
RuleEntry  (одна сущность на любой тип контента: Spell/Monster/Class/Race/Feat/...,
            Stats — JSONB, гибкая схема под категорию)
```

Это позволяет добавлять новые официальные или кастомные игровые системы без копипасты
вертикального среза Domain→Persistence→API→Web под каждую (см. фазу S, п. «другие системы»).

Ядро VTT/контента/RU закрыто. Живой список без устаревших пунктов:

- [ ] Качество RU: замена MT community-переводами по мере появления (актуально для D&D5e —
      см. фазу S, «допиливание D&D5e», где расписано конкретнее)
- [ ] Мультиязычность UI EN/RU (низкий приоритет, аудитория русскоязычная)

### Дальний ящик / упирается в лицензии (осознанно не берётся в работу)
- [~] Импорт сцен/токенов из Foundry world pack
- [~] Официальные компендиумы Paizo one-click
- [~] Лицензированный арт монстров

RU-контент, менять модель нельзя без пересмотра лицензий.

## Фаза S — Развитие продукта за пределами Foundry-паритета (открыто)

Паритет с Foundry по PF2e закрыт (фазы H–Q). Направления роста продукта дальше, осознанно
выбранные из более широкого списка идей (2026-07-15) — остальное пока не планируется вообще.

- [ ] **Другие игровые системы** — сейчас PF2e (полный) + D&D5e (черновой). Добавить ещё одну
      систему через уже готовый механизм `GameSystem.CreateCustom`/`RuleEntry`
- [ ] **Допиливание D&D5e** — сейчас помечено как MT-черновик (2026-07-15). Глоссарий терминов
      поверх машинного перевода (по аналогии с `Pf2eLocaleService` у PF2e), правка конкретных
      багов данных (перепутанный контент рас, транслитерация вместо перевода фитов)
- [x] **Тесты** (закрыто 2026-07-16) — 828 тестов. API-слой полностью покрыт через
      `WebApplicationFactory`; Web-слой впервые получил покрытие через bUnit — все основные
      паттерны страниц (create-формы, detail-fetch, списки с пагинацией, auth-gated загрузка,
      RU-locale-сервис). Осознанно не покрыто: Table.razor (SignalR-хаб + canvas/JS-интероп) и
      file-upload страницы (Tickets, Characters-импорт) — непропорциональные усилия для bUnit,
      не в приоритете. Дальнейшее расширение — по мере изменений кода, не отдельным проектом.
- [ ] **Мобильный опыт / PWA** — стол сессии физически нужен с телефона за игровым столом;
      проверить и доработать юзабельность на мобильном экране, рассмотреть PWA
- [ ] **AI-помощник — и для ГМа, и для игроков** — резюме сессии из чата/журнала, генерация NPC
      на лету, подсказки по правилам для ГМа; для игроков — помощь в билде, объяснение способностей
- [ ] **Аналитика/дашборд ГМа** — история группы, статистика посещаемости сессий
- [ ] **Соцфункции** — развить существующие форум/LFG/рейтинги: профили-портфолио ГМов,
      развёрнутые отзывы, значки/достижения
- [ ] **Observability/инфраструктура** — Seq-логи уже есть; алертинг, бэкапы БД, rate-limiting API
- [ ] **Онбординг** — развить `/start` в полноценный интерактивный тур

Осознанно не планируется (2026-07-15): монетизация, лицензированный контент/арт,
мультиязычность UI, импорт Foundry world pack/официальные компендиумы — зафиксированы в
«дальнем ящике» фазы R, но не в активной проработке.
